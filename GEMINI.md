# Sportsbook Project Instructions

## Workflow
- **Iteration Commits**: Always perform a git commit after successfully completing each iteration. The commit message should follow the format: "Iteration N: [Name] - [Brief description]".
- **Architecture**: Follow the modular monolith and package-by-feature structure as described in the project prompt.
- **Security**: Adhere to the four levels of protection: Spring Security, Object-level authorization, Business validation, and Fraud decision.

## Documentation
- **Project Context**: Maintain `docs/PROJECT_CONTEXT.md` with a summary of the current project state, recent changes, and pending items. Update it after completing each iteration.
- **Changelog**: Maintain `docs/CHANGELOG.md` with a list of changes per iteration.
- **Technical Documentation**: Maintain `docs/DOCUMENTATION.md` for detailed technical specifications.
- **UML**: Maintain `docs/UML.md` with pseudographic UML diagrams of key patterns and class structures.
